# Xcode Screenshot & Visual Analysis

Capture screenshots of the running app and analyze them using computer vision for visual debugging, UI verification, and regression detection.

## When to Use

- Verify UI changes after implementation
- Debug layout issues (spacing, alignment, overflow)
- Confirm theme/color application
- Check component rendering
- Visual regression detection
- Accessibility visual audits
- Before/after comparisons

## Process

### 1. Verify App is Running

```bash
if ! pgrep -x "SpuriousTime-macOS" > /dev/null; then
  echo "⚠️ APP NOT RUNNING"
  echo "Launch app first with /xcode-launch or /xcode-workflow"
  exit 1
fi
```

### 2. Setup Screenshot Directory

**Detect project root** (git repository root):
```bash
PROJECT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null)
if [ -z "$PROJECT_ROOT" ]; then
  echo "⚠️ Not in a git repository"
  PROJECT_ROOT="$PWD"
fi
```

**Create screenshot directory structure**:
```bash
DATE_DIR="$PROJECT_ROOT/.screenshots/$(date +%Y-%m-%d)"
mkdir -p "$DATE_DIR"

# Ensure .gitignore includes .screenshots/
if [ -f "$PROJECT_ROOT/.gitignore" ]; then
  if ! grep -q "^\.screenshots/$" "$PROJECT_ROOT/.gitignore"; then
    echo ".screenshots/" >> "$PROJECT_ROOT/.gitignore"
    echo "✅ Added .screenshots/ to .gitignore"
  fi
else
  echo ".screenshots/" > "$PROJECT_ROOT/.gitignore"
  echo "✅ Created .gitignore with .screenshots/"
fi
```

### 3. Capture Screenshot

**Generate filename** (timestamp + description):
```bash
TIMESTAMP=$(date +%H%M%S)
# Description provided as argument or default
DESCRIPTION="${1:-screenshot}"
FILENAME="${TIMESTAMP}-${DESCRIPTION}.png"
SCREENSHOT_PATH="$DATE_DIR/$FILENAME"
```

**Capture app window**:
```bash
# Active window
screencapture -x -o -w "$SCREENSHOT_PATH"

# OR specific window by PID
PID=$(pgrep -x "SpuriousTime-macOS")
screencapture -x -l $PID "$SCREENSHOT_PATH"

echo "📸 Screenshot saved: .screenshots/$(date +%Y-%m-%d)/$FILENAME"
```

**Timeout**: 5 seconds

### 4. Analyze Screenshot with Vision

Read the screenshot file using the Read tool, then create analysis notes.

### 5. Create Analysis Notes

**Create markdown file** alongside screenshot:
```bash
NOTES_PATH="$DATE_DIR/${TIMESTAMP}-${DESCRIPTION}.md"

cat > "$NOTES_PATH" <<EOF
# Screenshot Analysis: $DESCRIPTION

**Date**: $(date +%Y-%m-%d)
**Time**: $(date +%H:%M:%S)
**File**: $FILENAME
**Git Branch**: $(git rev-parse --abbrev-ref HEAD 2>/dev/null || echo "N/A")
**Git Commit**: $(git rev-parse --short HEAD 2>/dev/null || echo "N/A")

## Visual Analysis

[Analysis from vision model will be inserted here]

## Issues Detected

- [ ] Issue 1: Description
- [ ] Issue 2: Description

## Files to Review

- \`path/to/file.swift:line\` - Reason

## Status

- [ ] Reviewed by user
- [ ] Issues addressed
- [ ] Ready to archive

---
*Generated by xcode-screenshot skill*
EOF
```

### 6. Update Review Checklist

**Update master REVIEW.md**:
```bash
REVIEW_FILE="$PROJECT_ROOT/.screenshots/REVIEW.md"

# Create if doesn't exist
if [ ! -f "$REVIEW_FILE" ]; then
  cat > "$REVIEW_FILE" <<EOF
# Screenshots Review Checklist

## Unreviewed Screenshots

EOF
fi

# Add new entry
cat >> "$REVIEW_FILE" <<EOF
- [ ] $(date +%Y-%m-%d) $FILENAME - $DESCRIPTION
EOF

echo ""
echo "📋 REMINDER: Review screenshot at:"
echo "   .screenshots/$(date +%Y-%m-%d)/$FILENAME"
echo ""
echo "   Checklist updated: .screenshots/REVIEW.md"
```

### 7. Visual Analysis Checklist

When analyzing screenshots, check for:

**Layout Issues**:
- Text truncation or overflow
- Overlapping elements
- Inconsistent spacing
- Alignment problems
- Missing padding/margins
- Elements extending beyond bounds

**Color/Theme Issues**:
- Incorrect theme application
- Color contrast problems
- Missing theme tokens
- System colors instead of theme colors
- Inconsistent color usage

**Component Rendering**:
- Missing UI elements
- Broken icons or images
- Incorrect component state
- Animation artifacts
- Rendering glitches

**Accessibility**:
- Text too small
- Insufficient contrast
- Touch targets too small
- Missing visual feedback

**State Verification**:
- Correct data displayed
- Proper loading states
- Error states visible
- Empty states rendered

## Comparison Mode (Before/After)

For visual regression testing:

```bash
# Setup
PROJECT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null)
DATE_DIR="$PROJECT_ROOT/.screenshots/$(date +%Y-%m-%d)"
mkdir -p "$DATE_DIR"

# Capture "before" state
TIMESTAMP=$(date +%H%M%S)
BEFORE="$DATE_DIR/${TIMESTAMP}-before.png"
screencapture -x -o -w "$BEFORE"

# Make changes, rebuild, relaunch
# ... (code changes, xcode-workflow build and run)

# Capture "after" state
TIMESTAMP=$(date +%H%M%S)
AFTER="$DATE_DIR/${TIMESTAMP}-after.png"
screencapture -x -o -w "$AFTER"

# Create comparison notes
cat > "$DATE_DIR/${TIMESTAMP}-comparison.md" <<EOF
# Before/After Comparison

**Before**: ${BEFORE##*/}
**After**: ${AFTER##*/}

## Changes Made
[List code changes]

## Visual Differences
[Analysis from Read tool]

## Regression Check
- [ ] Intentional changes only
- [ ] No unexpected regressions
- [ ] User reviewed

EOF

# Compare visually (via Read tool analysis)
```

Report differences:
```
📊 BEFORE/AFTER COMPARISON

Before: .screenshots/2026-01-31/143022-before.png
After:  .screenshots/2026-01-31/143045-after.png

Differences detected:
- Button color changed from blue to accent color ✅
- Spacing increased by 8pt ✅
- Text now truncates with ellipsis ✅

📝 Comparison notes: .screenshots/2026-01-31/143045-comparison.md

📋 REMINDER: Review comparison at:
   .screenshots/2026-01-31/143045-comparison.md

Regression check: PASS
```

## Integration with Workflows

**After implementation**:
```
1. Skill("xcode-workflow") → "build and run"
2. Wait 3 seconds for app to stabilize
3. Skill("xcode-screenshot")
4. Analyze screenshot
5. Report visual verification results
```

**Visual regression workflow**:
```
1. Skill("xcode-screenshot") → Capture baseline
2. Make code changes
3. Skill("xcode-workflow") → "build and run"
4. Skill("xcode-screenshot") → Capture new state
5. Compare and report differences
```

## Error Handling

**App not running**:
```
⚠️ APP NOT RUNNING

Cannot capture screenshot of app that's not running.

Next: Launch app with /xcode-workflow build-launch
```

**Screenshot failed**:
```
❌ SCREENSHOT FAILED

screencapture returned error.

Possible causes:
- Screen recording permission denied
- Disk full
- Invalid window ID

Fix: Grant screen recording permission in System Preferences
```

**Cannot read screenshot**:
```
❌ CANNOT READ SCREENSHOT

Screenshot file exists but cannot be read.

File: /tmp/spurious-screenshot-*.png
Size: 0 bytes

Cause: screencapture failed silently
Fix: Retry screenshot capture
```

## macOS Permissions

**Required**: Screen Recording permission

Check permission:
```bash
# If permission denied, macOS will prompt automatically on first use
screencapture -x /tmp/test.png 2>&1
```

Grant permission:
```
System Preferences → Privacy & Security → Screen Recording
→ Add Terminal (or IDE running Claude)
```

## Directory Structure

All screenshots are stored in the project root:

```
<project-root>/
├── .gitignore                     (auto-updated with .screenshots/)
└── .screenshots/                  (git-ignored)
    ├── REVIEW.md                  (master checklist)
    ├── 2026-01-31/
    │   ├── 143022-theme-light.png
    │   ├── 143022-theme-light.md  (analysis notes)
    │   ├── 143055-theme-dark.png
    │   ├── 143055-theme-dark.md
    │   └── 144230-layout-issue.png
    └── 2026-02-01/
        ├── 091522-before.png
        ├── 091655-after.png
        └── 091655-comparison.md
```

**Organization**:
- By date: `YYYY-MM-DD/` folders
- Naming: `HHMMSS-description.png`
- Analysis: `HHMMSS-description.md` (same basename)
- Review: `REVIEW.md` at root with checklist

## Review Workflow

1. **Capture** screenshot during development
2. **Analyze** with vision and create notes
3. **Reminder** displayed in output
4. **Review** manually by opening `.screenshots/` directory
5. **Check off** items in `REVIEW.md` as reviewed
6. **Mark** analysis notes as reviewed

**REVIEW.md format**:
```markdown
# Screenshots Review Checklist

## Unreviewed Screenshots

- [ ] 2026-01-31 143022-theme-light.png - theme-light
- [ ] 2026-01-31 143055-theme-dark.png - theme-dark
- [ ] 2026-01-31 144230-layout-issue.png - layout-issue

## Reviewed (move items here after review)

- [x] 2026-01-30 120533-initial-test.png - initial-test
```

## Advanced Usage

**Annotated screenshots** (cursor visible, delay):
```bash
DATE_DIR="$PROJECT_ROOT/.screenshots/$(date +%Y-%m-%d)"

# Capture with cursor visible
screencapture -x -C "$DATE_DIR/screenshot.png"

# Capture after 5 second delay (time to arrange UI)
screencapture -x -T 5 "$DATE_DIR/screenshot.png"
```

**Multiple monitor support**:
```bash
DATE_DIR="$PROJECT_ROOT/.screenshots/$(date +%Y-%m-%d)"

# Display 1 only
screencapture -x -D 1 "$DATE_DIR/display1.png"

# Display 2 only
screencapture -x -D 2 "$DATE_DIR/display2.png"
```

**Cleanup old screenshots**:
```bash
# Find screenshots older than 30 days
find "$PROJECT_ROOT/.screenshots" -type f -mtime +30 -name "*.png"

# Delete after confirming
find "$PROJECT_ROOT/.screenshots" -type f -mtime +30 -name "*.png" -delete
```

- Screenshots saved to `<project-root>/.screenshots/` (git-ignored)
- Organized by date: `YYYY-MM-DD/` folders
- Analysis notes in markdown alongside screenshots
- Master checklist in `REVIEW.md` for tracking reviews
- Use `-x` flag for no sound, faster capture
- Retina displays: Screenshots are 2x resolution automatically
- Works on all macOS versions 10.14+
- Read tool can analyze PNG, JPG, PDF formats
- For UI tests, combine with xcode-test skill
- Screenshots persist across sessions for manual review
- Check off items in REVIEW.md after reviewing
- Cleanup old screenshots manually or with find command
